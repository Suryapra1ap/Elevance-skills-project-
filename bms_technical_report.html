01. Executive Summary

This technical report details the design, implementation, and deployment analysis of an enterprise-grade **4-Cell Lithium-Ion Battery Management System (BMS) and Protection Kernel**. Built on the high-performance ESP32 microcontroller, the system employs an event-driven, non-blocking asynchronous state scheduler to manage high-precision cell monitoring, localized human-machine interfaces (HMI), and bidirectional IoT telemetry.

Traditional BMS architectures often suffer from latency issues or system lockups when executing network transactions or screen refreshes. This project resolves those vulnerabilities by running all key tasks—such as safety trip processing, LCD diagnostics rotation, and Blynk IoT telemetry synchronization—using isolated, non-blocking execution blocks governed by hardware timers and millis() offsets.

The resulting system achieves sub-millisecond edge protection against overcharge, over-discharge, and multi-cell thermal/voltage imbalances while sustaining a reliable telemetry uplink and allowing remote, cloud-based emergency overrides.

2.1 Hardware Peripheral Configuration

The hardware architecture interfaces with several key diagnostic and safety peripherals:

  • Analog Front End (AFE): Simulated by four high-precision linear potentiometers configured as cell voltage emulators connected to ESP32 ADC1 channels (GPIO 32, 33, 34, 35).
  • Galvanic Load Isolation: Simulated by an active-high electromagnetic SPDT relay (GPIO 4), cutting off the main power rail immediately during a safety trip.
  • Localized Alerts: Consists of a continuous active piezo buzzer (GPIO 5) and a high-brightness fault LED (GPIO 18) to deliver immediate on-site auditory and visual danger alerts.
  • On-Site Display: A LiquidCrystal \(20\times4\) characters display with an integrated PCF8574 \(I^2C\) backpack (GPIO 21/22), enabling high-density data diagnostics with minimal wiring.

03. Mathematical Foundations & Analytics

To evaluate cell tracking health and degradation, the battery intelligence engine runs a continuous mathematical model comparing individual cells to the active average of the series pack.

3.1 Cell Deviation & Imbalance Formula

The cell imbalance percentage is calculated in real-time by checking the absolute deviation between the highest cell voltage (\(V_{max}\)) and the lowest cell voltage (\(V_{min}\)) against the pack mean voltage (\(V_{avg}\)):

\[ Imbalance (\%) = \frac{V_{max} - V_{min}}{V_{avg}} \times 100 \]

Where the running average (\(V_{avg}\)) is computed dynamically:

\[ V_{avg} = \frac{1}{N} \sum_{i=1}^{N} V_{i} \]

For this system configuration, \(N = 4\).

3.2 Dynamic Volumetric Scaling

The raw 12-bit ADC reads (\(0\) to \(4095\)) are converted to the physical scaling models in firmware using the following equation:

\[ V_{cell} = \left( \frac{\text{Raw ADC}}{4095.0} \right) \times V_{scale} \]

In the simulation, \(V_{scale} = 4.5\text{V}\), allowing the emulation of both standard Lithium-Ion cells (\(3.7\text{V}\) nominal, \(4.2\text{V}\) charge limit) and minor overcharge scenarios.

04. Operational State Machine

The core protection loop utilizes four distinct operating states represented by a firmware-level state vector:

State 0

NORMAL MODE

All cells are inside safe limits (3.00V - 4.25V). Imbalance sits below 5.0%. Relay stays closed and loaded.

State 1

DEGRADED WARNING

Triggered by float wire reads (<0.1V) or hardware rail saturation (>4.49V). Relay stays closed.

State 2

FAILSAFE PROTECTION

Instantly opens the load relay under overvoltage, deep discharge, or critical balance anomalies (≥10%).

State 3

SHUTDOWN LOCKOUT

Forced emergency override via Blynk Cloud V9 toggle. Open contacts lock out state engine loops completely.

5.1 Real-World Hardware Substitutions

  • Voltage Acquisition: In a physical system, placing direct microcontroller ADC pins across a series pack will destroy the pins due to common-mode voltage stacking. Physical deployment requires a dedicated Analog Front End (AFE) IC (such as the Texas Instruments BQ76952 or Analog Devices LTC6811). These ICs read differential cell voltages safely and transmit them to the ESP32 via galvanically isolated \(I^2C\) or SPI lines.
  • Power Isolation Switch: While a small mechanical relay works well for simulation, physical battery packs deliver hundreds of amperes. These require high-current solid-state switches, such as parallel N-Channel Power MOSFETs driven by dedicated gate-driver chips, or heavy-duty electromagnetic industrial contactors.
  • Physical Signal Conditioning: Low-pass RC filters must be integrated on each analog balance input lead to filter out high-frequency electromagnetic interference (EMI) coming from active electric motors or switching battery chargers.
